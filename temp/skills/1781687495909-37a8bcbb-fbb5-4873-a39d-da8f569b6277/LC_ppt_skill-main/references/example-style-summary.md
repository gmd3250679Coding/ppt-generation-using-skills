# 示例风格摘要

安装版默认不包含 `examples/` 大文件。本文件沉淀示例 deck 中可复用的风格经验, 供生成时直接参考。

## 总体气质

- 企业科技风, 信息密度中高, 页面克制清爽。
- 蓝白为主, 深蓝用于封面、章节页、重点结论页。
- 青绿和橙色只做少量强调, 不做大面积主色。
- 图文关系偏业务汇报, 不走互联网营销海报风。

## 常见页面

- 封面: 深蓝背景或白底蓝色装饰, 标题中左对齐。
- 目录: 3-5 个章节, 大号编号, 章节名短句化。
- 章节页: 大号 `01/02/03`, 一句章节说明。
- 能力页: 3-4 张卡片, 每张卡片一个能力、一个场景、一个价值。
- 流程页: 4-6 步横向流程, 每步标题短, 说明不超过两行。
- 架构页: 应用层、能力层、模型层、数据层或基础设施层。
- 案例页: 背景、方案、效果, 效果用指标大字报承接。
- 总结页: 3 点价值 + 3 阶段路径。

## 文案方式

- 标题尽量写成结论句: `让业务人员用自然语言完成指标分析`。
- 正文避免空话: 少用"全面赋能", 多写"权限可继承组织架构"。
- 没有真实数据时写 `示例`、`预估`、`待确认`, 不编造确定口径。

## 可复用业务术语

- 指标、维度、口径、语义层、数据源、血缘、权限、审计。
- 问数、归因、解释、订阅、预警、推荐、下钻、交叉分析。
- 知识库、智能体、工具调用、流程编排、人工审核、运营闭环。
- 私有化部署、租户隔离、数据脱敏、内容安全、日志追踪。
