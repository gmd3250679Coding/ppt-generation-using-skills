# 交付检查清单

## 内容

- 每页只有一个中心观点。
- 标题能独立表达结论, 不是只有名词。
- 客户方案和产品方案包含场景、方案、价值, 不只有功能清单。
- 数据有口径或来源; 不确定的数据标注为待确认。
- 目录章节和正文页标题一致。

## 视觉

- 主色保持浪潮/海岳蓝白科技风。
- 正文页不连续 6 页全白; 章节页或重点页要制造节奏。
- 中文标题不超过两行, 没有单字掉行。
- 字体以微软雅黑/Microsoft YaHei 为主, 没有混入不协调字体。
- 图表、卡片、表格对齐统一, 留白足够。
- 页脚、页码、logo 不被内容遮挡。

## PPTX

- 页面元素可编辑, 没有整页截图化。
- 图片不丢失, 不依赖外部链接。
- 表格文字投屏可读, 不低于 10 pt。
- 文件能正常打开, 没有兼容性报错。
- `unzip -t "<output.pptx>"` 通过。
- `node scripts/inspect-inspur-pptx.mjs "<output.pptx>"` 通过, 且主要正文页能检出文本层。
- 页数与大纲一致; 10 页左右需求通常控制在 8-12 页。
- 如果 contact sheet 因系统 Python 缺少 `PIL` 失败, 优先切换 Codex bundled Python, 不要把已生成 PPTX 误判为失败。

## HTML

- `<title>` 已替换。
- 每个 `<section>` 都有 `class="slide ..."` 和 `data-layout`。
- 只使用 `light`、`soft`、`blue` 三种页面主题。
- 图片路径存在且使用相对路径。
- 左右键、滚轮、触摸翻页可用。
- 运行 `scripts/validate-inspur-deck.mjs` 通过。
